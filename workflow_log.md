## Phase 1: Ideation and Planning

### Prompt used
> "I need help for my Deep Generative Models final project. The project must incorporate LLMs and/or Diffusion Models, use an agentic workflow, and produce a fully functional app with an interactive UI."

### Agent output
The agent analyzed the assignment requirements and proposed three possible directions. Given the constraint of a laptop with a local NVIDIA GPU but a preference for lightweight execution, the agent recommended an **API-based approach** combining both required technology types.

**Selected proposal:** *Story Illustrator* — a creative app where users enter a story theme, the LLM writes a 4-scene story, and a diffusion model illustrates each scene.

**Technology stack decided:**
- LLM: Big Pickle (GLM-4.6 MoE 355B) via OpenRouter — free, 200k context
- Diffusion: FLUX.1-schnell via HuggingFace Inference API — free tier, no local GPU
- UI: Gradio — fast to build, suitable for demos

**Why this stack:** Fully API-based inference means zero local GPU memory usage. Both technologies are required by the assignment. The creative domain makes for a visually compelling demo.

## Phase 2: Architecture Design and Task Decomposition

### Prompt used
> "Design the full system architecture for the Story Illustrator app and break it down into implementation tasks."

### Agent output
The agent produced a system architecture diagram showing:
1. Gradio UI (input) → LLM API → Scene Parser → Diffusion API → Gradio UI (output)

**Task decomposition:**
- `llm_client.py` — wrap OpenRouter API, include structured prompt, parse output
- `image_client.py` — wrap HuggingFace API, handle retries for cold model starts
- `app.py` — Gradio layout, wire pipeline together
- `requirements.txt`, `.env.example`, `README.md`

**Key architectural decision:** The LLM is prompted with a strict output format (`SCENE N: / STORY: / IMAGE:`) so the scene parser can reliably split story text from image prompts without complex NLP.

### Technical bottleneck resolved
**Problem:** HuggingFace free-tier models return HTTP 503 when cold (not loaded in memory).
**Solution:** The agent added a retry loop in `image_client.py` with exponential wait (`20s × attempt`) before retrying, up to 3 attempts. This handles cold starts transparently without crashing the app.

**Problem:** LLM output formatting can be inconsistent.
**Solution:** The agent added a `parse_story()` fallback that fills missing scenes with placeholder content, preventing the UI from breaking if the LLM deviates from the format.
